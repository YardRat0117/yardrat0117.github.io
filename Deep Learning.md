- 定义 
	- 通过构建和训练多层神经网络来模拟人类大脑的学习过程的机器学习子领域
- 深度学习与机器学习的区别  
	- 自动特征提取
	- 模型复杂度
	- 数据和算力需求
- 深度学习的应用领域
	- 计算机视觉
	- 自然语言处理
	- 语音处理
	- 游戏与强化学习
- 人工神经元 受生物神经元启发而构建的计算模型
	- 输入层 每个神经元接收多个代表特定特征值的输入信号，每个输入信号对应一个权重(Weight)
	- 权重 每个输入信号对应的可学习参数，在训练过程中不断调整，影响输入型号对神经元输出的影响程度
	- 加权和 $z = \sum\limits^{n}_{i = 1}w_ix_i+b$ 其中$b$是偏置项(Bias)，是一个额外的可学习参数，用于调整加权和的结果，使得模型可以更好地拟合数据
	- 激活函数 加权和通过激活函数进行非线性变换，使得模型可以处理复杂的模式和决策问题，常见激活函数如下
	- 输出 经过激活函数得出的输出，隐藏层神经元的输出会作为下一个神经元的输入，输出层的神经元会产生模型的预测结果
- 神经网络 一个完整的神经网络由多个神经元组成，并且分为以下几层
	- 输入层(Input Layer) 直接接受数据的原始特征，每个神经元对应一个特征
	- 隐藏层(Hidden Layer) 通过不断学习和调整权重，提取数据的特征，可能有若干个
	- 输出层(Output Layer) 根据任务需求产生最终的预测结果对于分类任务输出每个类别的概率；对于回归任务会输出连续值
- 激活函数 用于引入非线性，使神经网络能够处理复杂的模式
	- ReLU (Rectified Linear Unit) 输出范围$\left[0, +\infty\right)$，计算速度快 $$f(x) = max(0, x)$$
		- 优点
			- 可以有效解决梯度消失问题 
			- 再正数区域梯度恒为1，能够加快收敛速度
		- 缺点
			- 可能导致“死亡ReLU”问题，即某些神经元在训练过程中始终输出零，无法更新权重，造成这些神经元失效
			- 对于负输入造成信息丢失
		- 应用场景 CNN和DNN，目前最常用激活函数之一
	- Sigmoid 输出范围$(0, 1)$，函数呈现S型曲线 $$f(x) = \frac{1}{1 + e^{-x}}$$
		- 优点
			- 对于二分类任务可以将输出解释为概率
			- 函数的性质使得反向传播算法可行
		- 缺点
			- 梯度消失问题 输入值很大/小时导数接近于零，反向传播时梯度消失，训练速度变慢
			- 输出不以零为中心，可能导致训练不稳定
		- 应用场景 二分类问题的输出层，特别是二分类神经网络中
	- Tanh 输出范围$\left(-1, 1\right)$，函数呈现S型曲线 $$f(x) = \tanh(x)=\frac{e^x-e^{-x}}{e^x+e^{-x}}$$
		- 优点
			- 输出范围更广，在零附近输出均衡，能够更快收敛
			- 导数在输入值为零时最大，有助于缓解梯度消失问题
		- 缺点
			- 在较大或较小的输入值时仍会出现梯度消失现象
			- 在正负输入之间对称性较好，但计算相对复杂
		- 应用场景 隐藏层，特别是RNN中
- 前向传播 数据从输入层通过每一层的计算，逐层传递到输出层，最终生成预测结果的过程 $$a^{(l+1)} = f\left(W^{(l)} \cdot a^{(l)}+b^{(l)}\right) \quad \text{其中}W^{(l)}\text{为权重矩阵}$$
- 损失函数 用于计算网络的预测值与真实值之间的差异，目标是通过最小化损失函数来优化模型 $$L = \mathrm{Loss}\left(y_{\mathrm{true}}, y_{\mathrm{pred}}\right) \quad y_{\mathrm{true}}\text{是真实标签，} y_{\mathrm{pred}}\text{是网络的预测结果}$$
	- 回归任务 预测连续数值的任务
		- 均方误差(Mean Squared Error, MSE)
			- 最常用的回归损失函数，用来度量预测值与真实值之间的误差的平方和
			- 对异常点非常敏感
			- $$\mathrm{MSE} = \frac{1}{N}\sum\limits^{N}_{i=1}\left(y_i-\hat{y_i}\right)^2$$
		- 平均绝对误差(Mean Absolute Error, MAE)
			- 度量预测值与真实值之间的绝对误差
			- 更加robust，不像MSE那样对异常点过于敏感
			- $$\mathrm{MAE} = \frac{1}{N}\sum\limits^{N}_{i = 1}\left|y_i-\hat{y_i}\right|$$
	- 分类任务 预测离散类别的任务
		- 交叉熵模型(Cross-Entropy Loss, CEL) 最常用的分类损失函数，度量预测分布与真实分布之间的距离
			- $$\text{CEL} = -\sum\limits_{i=1}^{N}y_i\log \hat y_i$$
			  其中$y_i$是真实类别(一般是one-hot编码)，$\hat y_i$是预测概率
			- 对于二分类任务，交叉熵损失的公式为$$\text{Binary CEL} = -\frac1N\sum\limits_{i=1}^{N}\left[y_i\log{\left(\hat y_i\right)}+\left(1-y_i\right)\log{\left(1-\hat y_i\right)}\right]$$
			- 多分类任务则使用Softmax交叉熵损失，Softmax激活函数将网络输出原始得分转换为预测概率并确保输出服从概率分布$$\hat y_i=\frac{e^{z_i}}{\sum_{j=1}^{C}e^{z_j}}$$
			  其中$C$是类别数
		- Huber Loss
			- $$L_\delta\left(y, \hat{y}\right)=\begin{cases}\frac{1}{2}\left(y - \hat{y}\right)^2 & \text{if } \left|y - \hat{y}\right|\leq\delta \\ \delta\cdot\left|y - \hat{y}\right|-\frac{1}{2}\delta^2 & \text{if }\left|y - \hat{y}\right|\gt\delta\end{cases}$$
			  其中$\delta$是一个超参数，用来控制损失函数在MSE和MAE之间的切换点
			- 结合了MSE和MAE的优点，适用于对异常点有一定容忍度的回归任务
			- 误差较小时像MSE，但在误差较大时则像MAE，不那么敏感
	- 损失函数的选择 
		- 回归任务 使用MSE或MAE
		- 二分类任务 使用二分类交叉熵损失
		- 多分类任务 使用Softmax交叉熵损失
		- 异常值存在 Huber Loss 或 MAE 对异常值更robust
- 反向传播 从输出层开始，逐层将误差(即梯度)传递回前面的层通过链式法则，计算损失函数对每一层权重和偏置的偏导数(即梯度)，再使用这些梯度更新网络参数 
	- 计算输出层的梯度 计算损失函数相对于输出层的梯度 $$\hat y - y = \delta^{(L)} = \frac{\partial L}{\partial z^{(l)}} = \frac{\partial L}{\partial a^{(L)}} \cdot \frac{\partial a^{(l)}}{\partial z^{(l)}} = \frac{\partial L}{\partial a^{(L)}} \cdot f^{\prime}(z^{(l)})$$
	- 计算隐藏层的梯度 隐藏层的梯度通过输出层的梯度反向传播回去 $$\delta^{(l)} = \left(\delta^{(l+1)}\cdot W^{(l+1)}\right)\cdot f^{\prime}(z^{(l)}) $$
	- 更新权重与偏置项 通过梯度下降法或其改进算法更新每一层的权重和偏置项$$W^{(l)}=W^{(l)}-\alpha\cdot\frac{\partial L}{\partial W^{(l)}}$$$$b^{(l)} = b^{(l)} - \alpha\cdot\frac{\partial L}{\partial b^{(l)}}$$
	  其中学习率$\alpha$是一个超参数，用于控制每次更新的步长，过大可能导致模型不稳定，过小则可能导致收敛速度过慢
- 梯度下降
	- 目标 通过最小化损失函数更新模型的参数
	- 步骤
		- 初始化参数 随机选择参数的初始值
		- 计算梯度 在当前参数下计算损失函数关于参数的梯度
		- 更新参数 根据梯度更新参数使用以下公式 $$\theta = \theta - \alpha\cdot\nabla{(\theta)}$$
		  其中学习率$\alpha$为超参数，控制每次更新的步长过大可能导致模型不稳定，过小可能导致收敛速度过慢
	- 变种
		- 批量梯度下降(Batch Gradient Descent)
			- 使用整个训练数据集计算梯度，适合小数据集
			- 收敛稳定，但计算开销大，尤其数据量较大时
		- 随机梯度下降(Stochastic Gradient Descent, SGD)
			- 每次使用单个样本计算梯度
			- 计算速度快，适用于大数据集，但可能会在最小值附近震荡
		- 小批量梯度下降(Mini-batch Gradient Descent)
			- 每次使用一小部分样本(如32或64个)计算梯度，综合了批量和随机的优点
			- 加快收敛速度，并减少震荡
- 梯度下降的优化
	- Momentum
		- 在参数更新时考虑上一次的更新量，引入“动量”来加速收敛
			- $$v_t=\beta v_{t-1}+\left(1-\beta\right)\nabla{L(\theta)}$$ $$\theta=\theta-\alpha v_t$$
			  其中$\beta$为动量因子，通常取值在0.5到0.9之间
	- RMSprop
		- 自适应学习率方法，针对每个参数调整学习率，减少震荡
			- $$v_t=\beta v_{t-1}+\left(1-\beta\right)(\nabla{L(\theta)})^2$$ $$\theta=\theta - \frac{\alpha}{\sqrt{v_t+\epsilon}}\nabla{L(\theta)}$$
			其中$\epsilon$是为了防止除零错误的一个小常数，常取`1e-8`
	- Adam (Adaptive Moment Estimation)
		- 综合了Momentum和RMSprop的优点，使用一阶和二阶动量的估计
		- $$m_t=\beta_1m_{t-1}+\left(1-\beta_1\right)\nabla{L(\theta)}\quad\hat m_t=\frac{m_t}{1-\beta_1^t}$$ $$v_t=\beta_2v_{t-1}+(1-\beta_2)\left(\nabla{L(\theta)}\right)^2\quad\hat v_t = \frac{v_t}{1-\beta_2^2}$$ $$\theta = \theta - \frac{\alpha}{\sqrt{\hat v_t}+\epsilon}\hat{m_t}$$
		  其中$\beta_1$和$\beta_2$分别是一阶和二阶动量的衰减率(通常取值为0.9和0.999)
- 过拟合(Overfitting)
	- 特征
		- 训练集表现良好，测试集表现不佳
		- 对细微噪声较敏感
	- 原因
		- 模型过于复杂
		- 训练数据不足
		- 训练时间过长
	- 解决办法
		- 减少模型复杂度 减少层数或神经元数量
		- 正则化技术 L1/L2正则化或Dropout
		- 增加训练数据
		- 数据增强 对原始数据进行变换生成更多训练样本
		- Early Stopping 在验证误差开始上升时停止训练
- 欠拟合(Underfitting)
	- 特征
		- 训练集和测试集都表现较差
	- 原因
		- 模型过于简单
		- 特征不足或不适当
		- 训练不足
	- 解决办法
		- 增加模型复杂度 增加层数或神经元数量
		- 改进特征工程 提取更有代表性的特征，或增加更多有效特征
		- 优化训练过程 适当增加训练时间和次数，或调整超参数
- 正则化
	- 目的 惩罚模型的复杂度来改善模型的泛化能力，从而减少过拟合
	- L1正则化(Lasso)
		- 原理 通过向损失函数添加参数的绝对值和作为惩罚项来限制模型复杂度
		- $L_{\text{total}} = L_{\text{original}}+\lambda\sum\limits_i\left|w_i\right|$ 其中$\lambda$是正则化强度的超参数，$w_i$是模型的权重
		- 效果 L1正则化倾向于产生稀疏模型，即使得很多权重趋于零这在高维特征空间下可以帮助选出更重要的特性
		- 适用场景 怀疑输入特征中有许多不相关或冗余特征时
	- L2正则化(Ridge)
		- 原理 通过向损失函数添加参数的平方和作为惩罚项
		- $L_{\text{total}}=L_{\text{original}}+\lambda\sum\limits_i w_i^2$
		- 效果 L2正则化会惩罚较大权重值，使得模型的权重更加平滑、均匀会减小所有的权重，但不一定会使得权重完全为零
		- 适用场景 希望能够平滑地调整权重时
	- Dropout正则化
		- 原理 每次训练时随机“丢弃”一部分神经元，使得网络的不同子网络被训练具体操作是在每次迭代时以某个概率$p$讲一部分神经元的输出设为零
		- 效果 有助于避免神经元之间的共适应性，使得模型更robust
		- 适用场景 深度网络，尤其时模型参数较多时
	- Early Stopping
		- 原理 通过在训练过程中监控验证集的损失值，当损失不再下降或开始上升就提前停止训练
		- 效果 避免了模型过度拟合训练集数据
		- 使用场景 简单但有效的防止过拟合方法
	- 数据增强(Data Augmentation)
		- 原理 通过对训练数据进行各种变换(旋转、平移、缩放、裁剪、噪声等)来扩充数据集，从而增强模型的泛化能力
		- 效果 提供更多样化的训练样本
		- 使用场景 图像处理、语音处理等领域
- 模型评估
	- 目的 评估模型的性能，确保能够在训练集和测试集上都取得良好的泛化性能
	- 常见评估指标
		- 分类问题
			- 准确率 多用于比例相对平衡的情况 $$\text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}}$$
			- 精确率 多用于FP代价较高的情况 $$\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}$$
			- 召回率 多用于FN代价较高的情况 $$\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}$$
			- 特异性 多用于FP代价较高的情况 $$\text{Specificity} = \frac{\text{TN}}{\text{TN} + \text{FP}}$$
			- F1分数 多用于类别不平衡的情况 $$\text{F1} = 2\cdot\frac{\text{Precision}\cdot\text{Recall}}{\text{Precision} + \text{Recall}}$$
			- AUC-ROC
				- ROC(Receiver Operating Characteristic)曲线 不同阈值下模型的假阳性率(False Positive Rate, FPR)与真阳性率(True Positive Rate, TPR)的变化关系，横轴为FPR，纵轴为TPR
				- AUC(Area Under Curve) ROC曲线下面积，用来衡量分类模型的整体表现表现，AUC值越大表示分类器性能越好
				- 多分类问题 可以转化为多个二分类问题，对每个类别计算AUC后取平均
				- 优缺点
					- 优点 不依赖具体阈值，可以综合反映模型性能，尤其是样本类别不均衡情况
					- 缺点 若数据极度不平衡，可能无法真实反映模型实际表现，需要配合其他指标
			- 宏平均和微平均 用于多分类问题计算指标
				- 宏平均(Marco Average) 
					- 对每个类别分别计算指标，然后取平均
					- 不考虑类别样本数量，适合样本数量大致相同的情况
				- 微平均
					- 基于全体数据集总数量进行计算，相当于对所有类别的数量加权求和后计算评估指标
					- 适合样本数量不平衡的场景，较为灵活
			- 混淆矩阵
				- 预测类别-实际类别矩阵，对角线值表示正确分类数量，非对角线值表示误分类数量
				- 对多分类问题，使用$n \times n$矩阵即可
			- 对数损失(log-loss)
				- $$\text{Log-Loss} = -\frac{1}{n}\sum_{i = 1}^{n}\sum_{k = 1}^{K}y_{i, k}\log{\left(p_{i, k}\right)}$$
				  其中$n$是样本总数，$K$是类别总数，$y_{i,k}$是第$i$个样本的真实类别(是类别$k$时为1，否则为0)，$p_{i,k}$是模型对第$i$个样本预测为类别$k$的概率
				- 评估的是模型输出的概率分布与真实类别的距离，通常用于衡量多分类模型的性能
				- 模型输出的每个类别的概率分布越接近真实的分布，log-loss越低
		- 回归问题
			- 均方误差(Mean Squared Error, MSE) 适用于关注较大误差的情况 $$\text{MSE} = \frac{1}{n}\sum\limits_{i = 1}^{n}\left(y_i-\hat y_i\right)^2$$
			- 均方根误差(Root Mean Squared Error, RMSE) 保留与原单位一致的量纲 $$\text{RMSE} = \sqrt{\frac{1}{n}\sum\limits_{i = 1}^{n}\left(y_i-\hat y_i\right)^2}$$
			- 平均绝对误差(Mean Absolute Error, MAE) 适用于衡量总体误差 $$\text{MAE} = \frac{1}{n}\sum\limits_{i = 1}^{n}\left|y_i-\hat y_i\right|$$
			- 决定系数($R^2$) 衡量模型拟合数据的程度，越接近1解释力越好
	- 数据重采样
		- 交叉验证(Cross-validation)
			- K折交叉验证(K-fold Cross-validation)
				- 将数据集分为K个子集(称为”折“)，然后轮流作为验证集，最后评估结果取平均
				- 优缺点
					- 优点 适合中等数据量，能减少评估偏差
					- 缺点 计算开销大，因为需要训练多次
				- 常见取值 通常K取5或10
			- 留一法(Leave-One-Out Cross-validation, LOO-CV)
				- 每次保留一个样本作为验证集
				- 优缺点
					- 优点 适合小数据集，理论上最精确的交叉验证方法
					- 缺点 计算代价极高
			- 分层交叉验证(Stratified K-fold Cross-validation) 针对类别不平衡的数据集
		- Bootstrapping
			- 从原始数据集中随机有放回抽样生成训练集，剩余样本生成测试集，重复多次，最后评估效果取平均
			- 优缺点
				- 优点 适合小数据集，可以评估模型性能分布并提供置信区间
				- 缺点 某些数据点可能多次出现导致模型过于依赖某些样本
			- 使用场景 适合小数据集，尤其没有足够数据进行标准交叉验证时，可以提供置信区间(即模型评估不确定性)
		- 多次随机划分数据
			- 多次将数据集随机划分为训练集和测试集，通过多次划分减少偶然性
			- 优缺点
				- 优点 减少单词划分带来的评估误差，且计算量比交叉验证小，因为每次只需要进行一次寻来你
				- 缺点 无法充分利用所有数据进行训练，尤其是在数据量较小时信息浪费较多
			- 使用场景 适合大数据集，尤其不希望多次训练时，常用于模型开发初期，快速检查模型表现
- 超参数调整
	- 常见超参数
		- 学习率(Learning Rate)
			- 作用 控制每次权重更新的步长
			- 问题 过大可能导致无法收敛或跳过最优点，过小可能导致训练过慢或陷入局部最优
			- 常见策略 使用学习率衰减(如阶梯衰减、指数衰减、余弦衰减)逐步减小学习率
		- 正则化参数(Regularization Parameter)
			- 作用 控制复杂度，防止过拟合
			- 类型 L1正则化(Lasso回归)和L2正则化(Ridge回归)，决定惩罚项的权重
			- 调整目标 通过调节正则化强度找到合适的模型复杂度
		- 批次大小(Batch Size)
			- 作用 决定每次迭代进行权重更新的样本大小
			- 问题 较小批次可能引入更多随机性导致不稳定，较大批次则需要较多内存
			- 常见选择 小批次(如32、64)适用于梯度下降，较大批次(如128、256)可提高训练效率、		
		- 权重初始化方式
			- 作用 影响训练收敛速度和性能
			- 常见方法 
				- 零初始化(Zero Initialization)
					- 描述 所有权重初始化为零
					- 问题 每个神经元都相同，丧失学习能力
					- 适用场景 几乎不使用在神经网络中
				- 随机初始化(Random Initialization)
					- 描述 权重初始化为随机的小值，通常采用均匀分布或正态分布
					- 优点 打破神经元之间的对称性，使得不同神经元可以学习不同的特征
					- 问题 如果随机初始化值过大或过小，可能会导致梯度消失或梯度爆炸
				- Xavier初始化(Glorot初始化)
					- 描述 权重初始化遵循正态分布或均匀分布，且满足特定公式 $$W \sim \mathcal{N}\left(0, \frac{1}{n_{in}}\right)$$
					  其中$n_{in}$是输入神经元的个数
					   - 优点 适合使用Sigmoid或tanh激活函数的网络，能够让输入和输出的方差保持平衡，避免梯度消失或梯度爆炸
					   - 应用场景 经典的深度学习网络中
				- He初始化
					- 描述 基于ReLU或其变种激活函数的初始化方法，权重初始化遵循 $$W \sim \mathcal{N}\left(0, \frac{2}{n_{in}}\right)$$
					   - 优点 适用于ReLU、Leaky ReLU等非线性激活函数，能够有效避免梯度消失问题
					   - 应用场景 现代神经网络，如深度卷积神经网络(CNN)
				- LeCun初始化
					- 描述 专为使用tanh或Sigmoid激活函数的网络设计，初始化方式为 $$W \sim \mathcal{N}\left(0, \frac{1}{n_{in}}\right)$$
					- 优点 在某些网络结构中能改善训练收敛速度
					- 应用场景 较少使用，但在特定条件下表现良好
		- 隐藏层数和神经元数(Hidden Layers and Neurons)
			- 作用 影响神经网络的复杂度
			- 调整目标 找到适合当前任务的网络结构，层数过少可能无法捕获复杂模式，层数过多可能导致过拟合
		- 激活函数(Activation Function)
			- 常见激活函数 ReLU(较为常用)、Sigmoid、Tanh
			- 调整目标 不同激活函数适合不同任务
		- 优化算法
			- 作用 影响训练收敛速度和稳定性
			- 常见算法 SGD、Adam、RMSprop
			- 调整目标 选择适合数据和任务的优化算法，Adam通常表现较好
	- 超参数调整方法
		- 网格搜索(Grid Search)
			- 原理 列举所有超参数组合，逐一训练模型并评估效果
			- 优点 可以覆盖所有可能组合，保障找到全局最优解
			- 缺点 计算开销大，特别是在超参数组合多且模型复杂时
		- 随机搜索(Random Search)
			- 原理 在超参数空间中随机采样超参数组合进行训练和评估
			- 优点 效率高，适合超参数空间较大的情况
			- 缺点 无法保证找到最优解
		- 贝叶斯优化(Bayesian Optimization)
			- 原理 基于已有的实验结果预测新的超参数组合，通过构建代理模型减少试验次数找到更好的超参数
			- 优点 减少计算成本，尤其适合计算代价大的模型
			- 缺点 实现复杂度高
		- 学习率调度(Learning Rate Scheduling)
			- 原理 在训练过程中逐步调整学习率
			- 常用方法
				- 固定学习率
				- 衰减学习率
					- 指数衰减 随着训练进行，学习率逐渐按比例下降$$\eta_{t} = \eta_{0} \cdot e^{-\lambda t}$$
					- 分段衰减 根据预设的训练轮数或时间点，手动降低学习率
					- 自适应学习率 当模型再验证集上表现停滞时自动减小学习率
				- 循环学习率(Cyclic Learning Rate, CLR) 学习率在训练过程中在一个范围内周期性地增加或减少，防止陷入局部最优
				- Warmup 训练初期使用较小学习率，之后逐步增大以避免初期震荡，常与其他学习率调度结合使用
			- 优点 可以提高模型的收敛速度，避免过大学习率跳过最优解
- 模型调试技巧  
	- 常见问题  
		- 过拟合(Overfitting)  
			- 描述 模型在训练集上表现很好，但在验证集或测试集上表现较差，表现出对训练数据的过度依赖  
			- 解决方法   
				- 增加正则化(如L1/L2正则化、Dropout)  
				- 增加训练数据量(如数据增强)  
				- 减少模型复杂度(如减少网络层数或神经元数量)  
				- 使用提前停止(Early Stopping)避免过长时间训练  
		- 欠拟合(Underfitting)  
			- 描述 模型在训练集和验证集上都表现不好，未能捕捉到数据中的复杂模式  
			- 解决方法   
				- 增加模型复杂度(如增加网络层数或神经元数量)  
				- 调整超参数(如增大学习率)  
				- 提高特征的表达能力(如引入新特征或使用特征工程)  
		- 梯度消失和梯度爆炸(Vanishing/Exploding Gradients)  
			- 描述 反向传播中梯度逐渐消失或迅速增大，导致模型无法有效收敛
			- 解决方法   
				- 使用适当的权重初始化方法(如He初始化、Xavier初始化)  
				- 使用合适的激活函数(如ReLU、Leaky ReLU)  
				- 使用Batch Normalization进行批量归一化  
- 训练加速和稳定化技巧  
	- Batch Normalization  
	    - 作用 对每个批次的数据进行归一化，使得该层的输入数据均值为0，方差为1$$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$
	      其中，$\mu_B$ 是批次均值，$\sigma_B^2$ 是批次方差，$\epsilon$ 是小常数以防止分母为零   
		  - 应用场景 广泛用于深度神经网络中，尤其层数较多的情况下
	  - Gradient Clipping
		  - 作用 防止梯度爆炸，限制梯度的最大值以保证训练稳定性$$\theta = \begin{cases} \theta & \text{if } ||\nabla J(\theta)|| \leq c \\ \theta - \frac{c}{||\nabla J(\theta)||} \nabla J(\theta) & \text{if } ||\nabla J(\theta)|| > c \end{cases}$$
		    其中，$c$ 是设定的阈值   
		- 应用场景 常用于深度网络中，特别是处理长序列数据时
	- Warm Restarts
		- 作用 定期重置学习率，帮助模型跳出局部最优$$\eta_t = \eta_{min} + \frac{1}{2} (\eta_{max} - \eta_{min}) \left(1 + \cos\left(\frac{t - t_{restart}}{T_{restart}} \pi\right)\right)$$
		  其中，$\eta_{max}$ 和 $\eta_{min}$ 分别为最大和最小学习率，$t_{restart}$ 为重启周期，$T_{restart}$ 是重启的间隔   
		- 应用场景 常与余弦退火等学习率调度方法结合使用，适用于需要长时间训练的任务，可以在一定程度上提升最终性能
- 模型解释性
	- 重要性
		- 透明性 了解模型的决策路径，增强对其预测结果的信任
		- 调试 发现并修复模型中的潜在问题，如过拟合、错误的特征学习
		- 责任分配 敏感场景下模型解释性对于责任划分和决策的正当性非常重要
		- 法规要求 模型解释性对于责任划分和决策的正当性非常重要 
	- 可解释性模型的分类
		- 全局可解释性(Global Interpretability) 展示模型整体的行为，比如模型如何对不同的特征进行加权 
		- 局部可解释性(Local Interpretability) 解释某个特定样本的预测结果，帮助理解模型如何对单个样本做出决策 
		- 内生可解释性(Intrinsically Interpretable Models) 模型本身结构简单、易于解释，如决策树、线性回归等 
		- 后处理可解释性(Post-hoc Interpretability) 通过额外的分析和技术解释复杂模型的预测结果，如神经网络、随机森林等 
	- 常见模型解释技术
		- 可视化技术
		    - 特征重要性图 展示哪些特征对模型的预测影响最大 
		    - Partial Dependence Plots (PDPs) 显示某个特征对模型输出的平均影响 
		    - Individual Conditional Expectation (ICE) 展示某个特征对单个样本的影响 
		- 基于近似的解释方法
		    - LIME(Local Interpretable Model-agnostic Explanations) 通过线性模型近似复杂模型的局部行为，帮助解释特定样本的预测 
		    - SHAP(SHapley Additive exPlanations) 基于博弈论的特征重要性解释方法，能够公平地分配每个特征对预测的贡献 
		- 可解释模型方法
		    - 决策树 通过层级结构展示模型的决策路径 
		    - 逻辑回归 通过系数说明每个特征对预测的贡献 
		- 神经网络的可解释性
		    - Grad-CAM(Gradient-weighted Class Activation Mapping) 用于卷积神经网络，生成激活热力图，展示模型关注的区域 
		    - Integrated Gradients 通过累积梯度值解释特征对模型输出的影响
